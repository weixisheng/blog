# npm发包

我们在开发的过程中都会从 npm 上下载包，来帮助我们提高工作效率。如果把自己的一些方法封装成包，并发布，后续自己安装就方便了。

此文主要记录最近发包到 npm 的过程。@hishion_wei/uniapp-utils (opens new window)

# 调试

npm 包在发布之前，其他项目还无法直接安装使用。这时候可以使用 npm link关联。

进入npm-test目录，执行npm link关联到全局，相当于npm install npm-test -g，只不过此时指向的是本地库。

然后找一个需要用到这个包的项目 my-project，执行：

npm link npm-test
1

从node_modules下面能看出 npm-test 与其他npm 包有些许不同，前面有个链接符号。

准备好了之后，引用这个包，像其他包一样使用其方法，可以看到控制台输出3。

import add from 'npm-test'

let sum = add(1, 2)
console.log(sum)
1
2
3
4

说明包里面的方法正常使用了，还可以根据情况转义es6代码为es5。

经过开发调试阶段，确认正常之后，现在可以在 my-project 取消关联，并准备发布到 npm 了。

npm unlink npm-test
1

# 版本管理

standard-version

使用 standard-version (opens new window) 生成版本更新和日志

npm i standard-version -D
1

然后在 package.json 添加一个script，后续升级执行 npm run release 即可。

"scripts": {
 //  "release": "standard-version"
 "release": "standard-version && git push --follow-tags origin master && npm publish"
}

# 管理

  • 发布scope包

如果发布 scope 包，需要修改 package.json 的 name 为包含组织的包名，格式为：@somescope/somepackagename，例如 @vue/cli-service

npm publish --access public
1
  • 删除包

使用 npm unpublish 命令删除某个包或者包的某个版本。

删除某个版本之后，这个版本就不再可用，必须使用新版本号；删除整个包需要24小时后才能发布新版本

# 删除包的某个版本
npm unpublish [<@scope>/]<pkg>@<version>
# 删除全部
npm unpublish [<@scope>/]<pkg> --force
1
2
3
4
  • 废弃包

如果不用维护某个包，或者提醒使用者更新版本，可以标记某个包或者某个版本为废弃的

npm deprecate <pkg>[@<version range>] <message>

## example
npm deprecate my-thing@"< 0.2.3" "critical bug fixed in v0.2.3"
npm deprecate my-thing@1.x "1.x is no longer supported"
